Javascript and the DOM

What is Javascript?

If HTML is the content and structure, and CSS is how it all looks, then JavaScript is everything else. It handles how the viewer interacts with the webpage, what happens with information, any updates or responses to things happening.

Let's look at a game of Uno.
HTML would be the all the cards, hands, the discard and draw pile.
CSS is how everything looks, the colours on the cards, how big they are, the fonts of the numbers and style.
JavaScript would be the actions. It would keep track of where the cards are, how many are in each hand, turn order, order of play, how many someone has to pick up, check whether the card you've played is valid, it could sort the cards in your hand, pick up the card, pass the turn to someone else and all sorts.

What is control flow and what are loops?

JavaScript is read as a series of instructions that typically starts from the top and runs through to the bottom. This order of reading is known as control flow. The control flow can be altered or interrupted if something happens. For example, let's say you're making your favourite sandwich. Lettuce, tomato, cucumber, chutney, ham and cheese. You have an order to the way you make that sandwich.

When placing the cucumbers on your sandwich, you could pick up each slice one by one, but that would be slow. Instead, you could use a loop and pick each of them up into your hand, and then loop again and place each one from your hand to the sandwich. The efficiency here is gained in not moving your hands back and forth so many times. Instead, you pick them all up, move to the sandwich, then place them all. Done!

What is the DOM?

The DOM Tree

When a web browser shows you a web page, it's showing a browser's interpretation of the HTML, CSS and JavaScript code. The browser reads that code and builds the Document Object Model (DOM) as a blueprint of the website and uses that to show users the website. If we used TradeMe as an example, you might be looking at the photos of a listing, clicking through them to see them all. Each time you click to see the next photo, when you click and see the photo change, that's JavaScript using the DOM to show the next picture.

What's the difference between arrays and objects?

The structure of an array
Toolbox object example

Similarly, they are both ways of organising information. They differ, by how they organise data.

An array is like a queue at the supermarket and an object is like a tool box. In an array, the items are ordered in a line, one by one, and you count through the line to find its position. Just like in a queue, you're waiting in line and your line number is represented by an index.
order: [
  "whiteCoffee", 
  "blackCoffee", 
  "hotChocolate" 
]
In an object, the items are in different compartments like screws or sockets in the top and the hammer is below. To get the hammer, you need to open the box, access the bottom and identify the hammer.
order: [
  "whiteCoffee", 
  "blackCoffee", 
  "hotChocolate" 
]

What are functions?

A function is like a script or procedure you follow for a specific task, like filling up the sink before washing the dishes. When you run the function "fillSink()" you turn on the hot tap and wait till it's hot, next you put in the plug. As it's filling you squeeze in some dishwashing liquid.
Functions are helpful, because you don't want to rewrite the procedure every time you want to perform the exact series of instructions. Also, if someone else wants to help with the dishes, the procedure for filling the sink is already written, so they can follow the same one.
