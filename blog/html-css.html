HTML/CSS - What are the best practices associated with using classes vs. ids?

What are they?

They are two types of selectors used in CSS to apply styling rules to elements in HTML. There are more selectors, and together they help to provide design consistency throughout a website.

Specificity

Just briefly, Specificity is a scoring system used to determine the hierarchy of which CSS rules are determined when targeting the same HTML tag. By default, higher rules will be overridden by rules declared lower down unless a rule with a higher specificity is declared.

The hierarchy stands as follows:
Important! > inline > id > class > element

So between Class and ID, ID is a higher priority when CSS rules are being selected.

What is a Class?

The class selector identifies all elements with the same class name attribute and applies CSS rules to them. Widely used on divs to make this container tag more specific, but very commonly used on most elements too.

HTML: 
<p class="text-blue"> I'm a red because I used the ID </p>

CSS: 
.text-blue{color: blue;}

I'm a blue instead of white because I used the class selector

What is an ID?

An ID is a selector that is used by developers to target or identify a specific element. In fact, each element can only have one ID. If an element is given two ID's, it won't respect any ID ruling. ID's are noted This is useful because you can apply styling to just one element and it won't be affected by other class declarations.

HTML: 
<p class="text-blue" id="text-red"> I'm a red because I used the ID </p>

CSS: 
#blog-box-red {background-color: red;}

I'm a red because I used the ID

What's the difference?

Consider it like this. A school is your HTML page, classrooms in a school are your classes, and the names of children or staff in each classroom are the IDs. The class Room 4 has an allocated time in the gym before it's Room 5's turn. But a student, Tim Buckland, is needed in the Principal's office. The class goes to the gym, but Tim goes to the principal's office because he specifically has been called, so it overrides his need to go to the gym. A class identifies a selection of elements with the class name and an ID is specific and identifies one element.

In Summary

It is best practice to use an ID once per page to preserve it's use and identify elements specifically. If more than one element must be selected, take advantage of the cascading nature of CSS and declare a class underneath other classes.

It should be noted that an ID can be used more than once on a page, despite my research indicating it couldn't.
